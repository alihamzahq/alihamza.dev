# Progress Tracking

## Development Workflow

**Branch Strategy**: `develop` → `feature/*` → commit → push → merge into `develop`

### Workflow Steps (Per Phase)

1. **Before starting**: Show phase summary + suggest feature branch name
2. **After branch approval**: Create feature branch from `develop`
3. **During development**: Implement tasks, no commits yet
4. **After phase complete**: Ask for review
5. **After review approval**: Show one-line commit message (no "claude" keywords)
6. **After commit approval**: Push to feature branch, merge into `develop`

### Rules

- One commit per phase only
- No keywords like "claude" or "generated by claude" in commits
- Always follow: `develop` → `feature/*` → commit → push → merge into `develop`
